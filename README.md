## 整体架构规范

本项目遵循开发规范进行分层，确保代码整体条理清晰：

| 层次  | 包名             | 职责描述                    |
|:----|:---------------|:------------------------|
| 表现层 | ui             | 负责所有控制台交互、菜单渲染及用户输入校验   |
| 服务层 | services       | 处理核心业务逻辑，协调仓储层与模型层      |
| 仓储层 | repository     | 负责数据的底层存储操作（内存存储或持久化存储） |
| 模型层 | models / enums | 定义业务实体对象与标准常量           |
| 工具层 | utils          | 提供日期处理等通用支持工具           |

## 如何运行

项目现在支持通过 `./src/main/java/Main` 类直接进行交互式引导选择周数，无需手动切换目录。

1. 环境准备：确保本地安装了 JDK 17 或更高版本。
2. 克隆仓库：
```bash
git clone https://github.com/mlfc666/CommonEnv.git
```
(或者直接从workflows自动构建中下载并运行jar包)
```bash
java -jar CommonEnv-SNAPSHOT.jar
```
