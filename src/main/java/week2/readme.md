# SQL演示系统 - 开发文档

## 1. 任务进度概览

### 1.1 核心数据库架构
- [x] **物理表结构自动化**：程序启动引导时自动校验并创建 `students`、`courses`、`scores` 核心三张表。
- [x] **严苛数据约束**：
  - **数值约束**：学生年龄 (15-25)、课程学分 (1.0-5.0)、考试成绩 (0.0-100.0) 均在数据库层实现 CHECK 约束。
  - **唯一性约束**：课程名称、学生手机号实现唯一性校验。
- [x] **物理关联引用**：通过外键关联学生与课程，并配置 `ON DELETE CASCADE` 级联删除，确保数据完整性。

### 1.2 业务功能实现
- [x] **多维数据管理**：实现学生、课程、成绩的动态录入、多条件精确更新与批量数据清理。
- [x] **进阶统计分析**：
  - 实现按班级分组的男女生人数“行转列”统计报表。
  - 实现基于教师维度的跨表关联成绩排名分析。
- [x] **复杂条件检索**：
  - 支持基于多字段组合排序（如班级升序+年龄降序）的过滤查询。
  - 支持基于双关键字通配符（LIKE）的学生成绩详情模糊查询。
- [x] **健壮性保障**：业务层集成自定义异常处理机制，对违规分值录入进行实时拦截。

## 2. 项目类层级说明

### 2.1 核心模型与传输对象 (Domain Models & DTOs)
| 类名称 | 职责说明 |
| :--- | :--- |
| `Student` | **学生实体**：映射数据库学生表，管理个人基础信息及班级属性 |
| `Course` | **课程实体**：映射课程表，管理教师、课程名及高精度学分 |
| `Score` | **成绩实体**：核心关联模型，记录考试时间、分数及性质备注 |
| `ClassGenderCountDTO` | **统计对象**：承载班级性别比例分析结果的视图模型 |
| `ScoreDetailDTO` | **详情对象**：承载多表联查（学生+课程+成绩）后的平面化视图数据 |

### 2.2 持久化层 (Repository Layer)
| 类/接口名称 | 职责说明 |
| :--- | :--- |
| `StudentRepository` | 管理学生表的所有 SQL 逻辑，包含性别筛选、年龄批量更新及性别分布统计 |
| `CourseRepository` | 管理课程表的所有 SQL 逻辑，负责课程信息维护及学分去重检索 |
| `ScoreRepository` | **核心逻辑中心**：负责多表 JOIN 查询、级联删除模拟以及成绩精确修正 |

### 2.3 业务逻辑层 (Service Layer)
| 类/接口名称 | 职责说明 |
| :--- | :--- |
| `StudentService` | 封装学生业务，处理分页偏移量计算、年龄增长规则及排序逻辑 |
| `CourseService` | 封装课程业务，管理课程准入规则及学分梯度筛选 |
| `ScoreService` | **规则校验中心**：负责分数范围合法性校验，并协调多表删除的事务一致性 |
| `StatisticsService` | **决策支持中心**：负责跨实体的聚合分析逻辑，生成统计报表数据 |

### 2.4 表现层与基础设施 (UI & Infrastructure)
| 类名称 | 职责说明 |
| :--- | :--- |
| `ConsoleMenu` | **交互控制层**：负责控制台菜单导航、用户输入捕获及彩色交互渲染 |
| `TablePrinter` | **视图渲染工具**：提供标准化的网格表格打印逻辑，支持各种 DTO 的可视化输出 |

| `JdbcUtils` | **连接连接管理**：基于环境变量适配的连接工厂，支持 Docker 部署动态配置 |
| `Main` | **引导类**：项目入口，负责各层组件的依赖注入与系统生命周期调度 |

## 3. 操作反馈说明

### 3.1 数据完整性保障
* **物理约束**：当用户录入不符合 CHECK 约束（如成绩 101 分）或 UNIQUE 约束（重复课程名）的数据时，系统会直接透传数据库底层的错误信息并终止操作，防止脏数据入库。
* **级联逻辑**：在删除一个学生或课程时，与其关联的所有历史成绩记录将自动从物理层面被清除，无需人工干预。

### 3.2 统计分析反馈
* **分组查询**：系统通过 `StatisticsService` 处理复杂的 SQL 聚合逻辑，能实时计算出各班级男女生人数，并以对齐表格形式反馈给管理人员。
* **关联分析**：支持通过任课教师姓名，瞬间检索出其名下所有学生的成绩详情及排名，数据跨越 `students`、`courses`、`scores` 三张物理表。

### 3.3 检索体验
* **模糊搜索**：支持对学生姓名进行非精确匹配，通过 `%关键字%` 逻辑，即使输入部分文字也能找回完整的考试记录及关联的科目信息。
